# 错误处理扩展功能说明

## 一、扩展功能概述

根据实验要求，扩展功能包括三项核心能力 [4]：

> "分析程序可以对"源程序"中的一些错误做出更多地的处理。如：错误的定位，续编译，简单错误的纠正等。" [4]

本次扩展对上述三项要求均进行了完整实现。

## 二、功能1：错误定位

### 2.1 实现原理

`Token.java` 新增 `line` 和 `column` 两个字段，记录每个 token 在源程序中的起始行号和列号。`Lexer.java` 新增 `advancePos(n)` 方法，在消费字符时自动追踪换行和列偏移，确保每个 token 携带精确位置。

### 2.2 错误输出格式

```
语法错误 [行X列Y]: <错误描述> （当前token: <类型> '<词素>'）
```

### 2.3 示例

对于输入 `begin x = ; y = 2 end`（等号后缺表达式），输出：

```
语法错误 [行1列12]: 因子错误，期望 id、整数或括号表达式 （当前token: SEMI ';'）
```

## 三、功能2：续编译

### 3.1 实现原理

旧的 `error()` 方法直接抛出 `RuntimeException` 终止分析。新的实现改为：
- 将错误消息记录到 `errorMessages` 列表中
- `errorCount` 自增
- 仅当错误数超过 `MAX_ERRORS`（默认10）时才终止
- 分析结束后在 `parseProgram()` 末尾统一输出错误汇总

### 3.2 错误汇总格式

```
========== 语法分析完成 ==========
共发现 N 个错误：
  [1] 语法错误 [行X列Y]: ...
  [2] 语法错误 [行X列Y]: ...
===================================
语法分析完成，但有 N 个错误。
```

### 3.3 恢复策略

| 恢复方式 | 触发位置 | 机制 |
|---------|---------|------|
| `match()` 简单纠正 | 期望 token 不匹配 | 尝试隐式补全（分号、括号、end/then），失败则跳过当前 token |
| `skipToSync()` 恐慌模式 | `parseF()` 等深层结构 | 跳到分号、end、else、then、do 等安全点 |
| 占位值返回 | 非法 token（ILOCT/ILHEX/UNKNOWN） | 返回 `ExprAttr("0")` 占位，避免 NPE |

## 四、功能3：简单错误纠正

### 4.1 四种纠正规则

| 规则 | 触发条件 | 行为 |
|------|---------|------|
| **缺分号** | 期望 `SEMI`，当前为 IDN/IF/WHILE/BEGIN/END/EOF | 当作隐式分号，不消耗 token |
| **缺右括号** | 期望 `SRP`，当前为 SEMI/END/ELSE/THEN/DO/EOF | 当作隐式右括号 |
| **缺 end** | 期望 `END`，当前为 EOF/SEMI/ELSE | 隐式补全 end |
| **缺 then** | 期望 `THEN`，当前为 IDN/IF/WHILE/BEGIN/END/EOF | 隐式补全 then |

### 4.2 典型场景

```
输入: begin x = 1; end
           ↑ 末尾缺少分号，直接遇到 end

旧行为: 报错"期望 SEMI，但遇到 END" → RuntimeException → 停止
新行为: 缺分号纠正 → 隐式分号 → 继续分析 → 输出语法树 + 1条警告
```
